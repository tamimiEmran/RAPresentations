Legend & key implementation notes

blue = data prep / loaders    green = outputs / saved artifacts    orange = LLM call    purple = prompt / dynamic substitution    red = error / OOM paths

  • Prompt-as-contract: the wrapper does not validate the LLM's JSON keys or values. classify.txt defines the schema; parsed_json stores whatever the model emits. prompt_sha in the parquet pins the exact prompt version that produced each row.
  • Substitution: render_prompt replaces {week_range} and {test_day} only — literal braces in the JSON schema example are preserved (regex-based, not str.format).
  • Per-day stats — ML parity: format_per_day_stats emits min/max/std/mean per day using np.nanmin/nanmax/nanstd/nanmean — identical to regenerate_dataset.py's q_stats and ref_stats features. Stats are included in all modes (text/image/both).
  • Modality flag: text = numeric series + stats + metadata, no chart;  image = chart + stats + metadata, no raw numbers;  both = chart + numeric series + stats + metadata.
  • Resume-on-failure: filter_done_samples skips rows already in the parquet whose error IS NULL. Errored rows (e.g. OOM) are retried; bad-JSON rows are not (parse_error set, error null) since the model would emit the same bad output.
  • Multi-model swap: --model qwen / deepseek-vl2 / llama-vision, or any full HuggingFace id. The chosen alias is stored verbatim in the model_alias column for grouping in downstream analysis.
  • Known parity gap: ML also has cal_month / dow / dom temporal features computed from real calendar dates in the source HDF5. The CSVs the LLM data loader currently reads do not include those timestamps. Closing this gap would mean switching the LLM data source to contextual_set/query_days.parquet (the same parquet ML uses).
